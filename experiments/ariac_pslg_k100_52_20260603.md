# ARIAC PSLG Variants - 100 Train / 52 Test - 2026-06-03

This follow-up repeats the PSLG/listwise/heatmap variants with more training
data:

```text
usable samples after duplicate-part filtering: 152
train: 100
test: 52
split_seed: 42
init_seed: 42
DINOv3 H+640 frozen dense features
d_slot: 256
coords: enabled
support head: two_stage
epochs: 120
```

The goal is to check whether the negative 52-train results were caused by too
little training data.

## Results

| setting | loss / extractor change | EM | F1 | top1 / top3 / top10 | missed stack | location region | wrong support |
| --- | --- | ---: | ---: | --- | ---: | ---: | ---: |
| H+640 baseline | CE + 0.2 structured + 0.2 CF | 0.8462 | 0.9723 | 0.9415 / 0.9766 / 1.0000 | 3 | 4 | 3 |
| listwise-heavy | 0.2 CE + 1.0 structured + 1.0 topK + 0.2 CF | 0.8269 | 0.9654 | 0.9240 / 0.9591 / 0.9942 | 3 | 7 | 3 |
| iterative + latent grounding 0.1 | baseline + loc/on latent grounding | **0.8654** | 0.9711 | 0.9357 / 0.9649 / 1.0000 | 3 | 6 | 2 |
| heatmap-only | heatmap query extractor, baseline loss | 0.5962 | 0.9264 | 0.8187 / 0.9474 / 0.9942 | 7 | 20 | 1 |
| iterative + loc-only latent 0.05 | baseline + loc-only latent grounding | 0.8269 | 0.9574 | 0.9123 / 0.9825 / 1.0000 | 4 | 8 | 3 |

## Interpretation

More training data does not rescue the most aggressive changes.

```text
listwise-heavy: 0.8462 -> 0.8269
heatmap-only:   0.8462 -> 0.5962
small topK:     0.8462 -> 0.8269
loc-only:       0.8462 -> 0.8269
```

The only EM improvement is:

```text
iterative + latent grounding 0.1:
  EM 0.8462 -> 0.8654
```

However, the improvement is one image on a 52-image test set. It is not a clean
ranking improvement:

```text
F1:             0.9723 -> 0.9711
placement top1: 0.9415 -> 0.9357
placement top3: 0.9766 -> 0.9649
location_region: 4 -> 6
wrong_support:   3 -> 2
```

So the latent grounding term may help a specific wrong-support case, but it also
worsens region ranking and global placement ranking.

## Conclusion

The 100-train run gives a weak positive signal for the full latent grounding
auxiliary, but not enough to make PSLG the main method.

Current conclusion:

```text
1. Naive heatmap query extraction should be rejected.
2. Heavy decoder-in-the-loop listwise training should be rejected in this form.
3. Loc-only latent grounding is not useful.
4. Full latent grounding 0.1 is worth keeping only as a tentative ablation,
   because it improves EM by one sample under 100 train but degrades F1/top-k
   ranking and increases location_region errors.
5. The main method should remain the simpler H+640 object-query two-stage
   structured+CF baseline unless multi-seed/split testing confirms the +1 image
   EM improvement.
```
