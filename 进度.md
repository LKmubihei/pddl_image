# ARIAC PDDL Image Grounding 进度总结

最后更新：2026-06-02

项目目录：`/home/pc/pddl_image`

## 1. 当前任务定义

目标是从 ARIAC 场景图像预测 PDDL `(:init ...)` 初始状态。

当前只使用真实 `(:init ...)` atoms 作为监督，不使用 goal，也不使用 VL conversation 文本。输出必须是合法 symbolic init state，主要 predicates 包括：

```text
part_at(part, location)
on(part, part)
clear(part)
handempty
robot_at(table)
```

当前核心建模方式不是直接独立预测所有 atoms，而是预测 placement factor：

```text
place(part) in {other active part, table, placement region}
```

然后由 deterministic decoder 推导：

```text
place(part) = location -> part_at(part, location)
place(part) = other_part -> on(part, other_part)
no part directly on p -> clear(p)
```

这样可以把 atom-level multi-label prediction 转成更符合 PDDL 结构的 support/location assignment 问题。

## 3. Frozen DINOv3 提供什么

当前主线使用 frozen DINOv3 H+ 640 dense features：

```text
image
-> DINOv3 H+ /16 at 640x640
-> 40 x 40 = 1600 dense patch tokens
-> 1280-d feature per token
-> append spatial coords
-> 1290-d token
```

主特征缓存保留在：

```text
experiments/ariac_52_100_hplus640_d256_coords_structcf_20260602/ariac_dinov3_raw_s640_l1_last_coords_features.pt
```

DINOv3 本身只提供匿名 dense visual tokens。它不直接知道：

```text
blue_battery
green_pump
regulator_placement
part_at
on
clear
```

所以后续小模块的任务不是重新学习视觉世界，而是把 frozen dense visual prior 适配到 PDDL symbol / candidate / legal state 空间。

## 4. 小模块的正确定位

当前结论非常明确：

```text
小模块 = frozen DINOv3 到 PDDL 结构空间之间的低参数、强归纳偏置 adapter
```

它不应该被定位成：

```text
小型 detector
小型 mask learner
小型 geometry learner
复杂 contact NN
大 scorer / 大 reranker
```

在 30-52 张训练样本下，这些复杂模块普遍不稳定。

当前应保留的小模块是：

```text
1. feature projector
2. PDDL object/location query extractor
3. 1-layer object relation layer
4. spatial coordinate features
5. two_stage placement scorer
6. structured legal-state loss
7. counterfactual margin loss
8. legal PDDL decoder
```

方法主线可以表述为：

```text
frozen DINOv3 H+640 dense features
+ PDDL-conditioned object queries
+ lightweight placement factor scorer
+ structured / counterfactual training
+ legal PDDL decoder
```

--feature-source dinov3_raw

## 6. 关键实验结果

### 6.1 Few-shot curve, same 100 test split

同一 100 张测试集，训练样本数变化：

| train K | EM | F1 | placement top1/top3/top10 | mean/max rank | missed_stack | location_region |
| ---: | ---: | ---: | ---: | ---: | ---: | ---: |
| 30 | 0.6100 | 0.9201 | 0.8138 / 0.8739 / 0.9910 | 1.874 / 12 | 11 | 44 |
| 40 | 0.6300 | 0.9190 | 0.8258 / 0.9099 / 0.9730 | 1.715 / 13 | 13 | 39 |
| 50 | 0.7500 | 0.9484 | 0.8919 / 0.9279 / 0.9940 | 1.399 / 13 | 8 | 19 |
| 52 | 0.7900 | 0.9603 | 0.9129 / 0.9700 / 0.9940 | 1.267 / 13 | 7 | 17 |

重要解释：

```text
0.7900 是 K=52 full train-pool 结果。
0.7500 是 strict K=50 结果。
```

K=50 相比 K=52 少掉的两个训练样本是：

```text
picture_374:
  red_battery -> battery_placement
  其他多个 part -> table

picture_74:
  blue_battery -> table
  blue_pump -> table
  green_regulator -> table
```

100 张 test 上 EM 差 0.04 等价于 4 张图，小样本下这种差异不能忽略。

### 6.2 Backbone / resolution 结果

H+640 相比 H+448 有稳定小幅收益：

| config | EM | F1 | location_region |
| --- | ---: | ---: | ---: |
| H+448 d256 coords structured+CF | 0.7700 | 0.9512 | 21 |
| H+640 d256 coords structured+CF | 0.7900 | 0.9603 | 17 |

结论：

```text
有效提升主要来自更好的 dense visual evidence / spatial resolution，
不是来自更复杂的 trainable scorer。
```

### 6.3 100 train / 52 test 参考结果

之前也跑过 100 train / 52 test：

```text
H+640 d256 coords structured+CF:
EM 0.8462
F1 0.9689
top1/top3/top10 0.9298 / 0.9649 / 1.0000
missed_stack 4
location_region 6
```

但这个 split 与当前 100-test few-shot curve 不同，只作为参考，不和 30/40/50/52 curve 直接混比。

## 7. 已尝试并排除的复杂小模块

这些模块目前不应进入主方法，最多放 appendix / negative ablation。

### 7.1 d_slot=512

H+448 下增大 slot 维度从 256 到 512 反而下降：

```text
H+448 d256: EM 0.7700
H+448 d512: EM 0.6600
```

说明少样本下增加 trainable capacity 会伤害 generalization。

### 7.2 typed_two_stage

把 scorer 拆成：

```text
part_scorer
location_scorer
candidate_bias
stack_gate
```

结果在 52 train / 100 test 下下降：

```text
two_stage baseline:
EM 0.7900
F1 0.9603
location_region 17

typed_two_stage:
EM 0.7200
F1 0.9502
location_region 24
```

结论：问题不是 scorer MLP 不够大，而是 scorer 没有稳定的空间 grounding evidence。拆 head 增参会过拟合。

### 7.3 calibration / dynamic hard negative / reranker

50 train / 100 test 下：

| config | EM | F1 | top1 | location_region |
| --- | ---: | ---: | ---: | ---: |
| baseline two_stage | 0.7500 | 0.9484 | 0.8919 | 19 |
| calibrated | 0.7500 | 0.9459 | 0.8739 | 23 |
| dynamic hard 0.2 | 0.7500 | 0.9507 | 0.8829 | 26 |
| calibrated + dynamic 0.2 | 0.8000 | 0.9585 | 0.9009 | 18 |
| calibrated + dynamic 0.1 | 0.7300 | 0.9484 | 0.8799 | 24 |
| calibrated + dynamic 0.5 | 0.6900 | 0.9364 | 0.8589 | 22 |
| + linear legal reranker | no change | no change | no change | no change |

虽然 `calibrated + dynamic 0.2` 在 strict K=50 上 EM 到 0.80，但它不能直接和 K=52 baseline 的 0.79 比；同时权重敏感，F1/top1/loc 错误没有稳定优势。因此不作为主方法。

### 7.4 attention geometry / location prior / local refinement

这些都基于 query attention 或 early top-k attention：

```text
attention geometry
location prior from attention center
local query refinement
```

结果均为负收益。结论：

```text
当前 query attention 不能当作可靠 object mask / bbox / center 使用。
```

### 7.5 patch-level candidate evidence

尝试了：

```text
base two_stage score
+ location patch evidence
+ optional table residual
+ optional support contact patch-pair evidence
```

50 train / 100 test：

| config | EM | F1 | top1/top3/top10 | missed_stack | location_region |
| --- | ---: | ---: | ---: | ---: | ---: |
| baseline two_stage K50 | 0.7500 | 0.9484 | 0.8919 / 0.9279 / 0.9940 | 8 | 19 |
| patch location scale0 | 0.7200 | 0.9431 | 0.8679 / 0.9520 / 0.9970 | 6 | 27 |
| patch location scale0.1 | 0.7600 | 0.9467 | 0.8739 / 0.9550 / 0.9970 | 10 | 27 |
| patch location+table scale0.1 | 0.7400 | 0.9442 | 0.8709 / 0.9489 / 0.9940 | 5 | 28 |
| patch loc+table+contact scale0.1 | 0.7000 | 0.9424 | 0.8649 / 0.9399 / 0.9970 | 7 | 27 |

结论：

```text
当前 PDDL-query-to-patch similarity 还不是可靠 localization signal。
patch evidence 提高部分 top3/top10，但没有改善 top1，location_region 反而变差。
```

## 8. 当前主要错误与诊断

主要错误集中在：

```text
region/table grounding
stack/contact grounding
support candidate ranking
```

少样本下最明显的问题是 `location_region`：

```text
K=52: location_region 17
K=50: location_region 19
K=40: location_region 39
K=30: location_region 44
```

同时 top10 通常还很高，说明：

```text
gold support/location 经常已经在候选里，
但 top1/top3 排序不稳。
```

这不是 decoder 合法性问题，decoder 已经保证 legal=1.0。瓶颈在：

```text
DINO dense visual features -> PDDL object/location grounding -> support/location ranking
```

## 9. 论文/方法叙事建议

不要把方法写成：

```text
我们训练多个神经模块学习 grounding。
```

应该写成：

```text
We freeze the dense visual backbone and train a lightweight PDDL-conditioned adapter.
The adapter does not learn generic perception; it binds dense visual tokens to named PDDL objects,
factors output into placement assignments, and decodes under symbolic legality constraints.
```

中文表述：

```text
我们冻结视觉 backbone，只训练一个轻量 PDDL 条件适配器。
这个适配器不学习通用视觉，而是把 dense visual tokens 绑定到已知 PDDL 对象和候选 support 上，
并通过 placement factorization 与 legal decoder 降低输出空间复杂度。
```

核心贡献应聚焦：

```text
1. frozen DINOv3 dense visual prior
2. PDDL-conditioned object queries
3. placement factorization
4. structured / counterfactual legal-state training
5. constrained symbolic decoding
```

负结果可作为 appendix 支撑：

```text
few-shot PDDL grounding 中，增加 trainable scorer capacity 或从未监督 attention 中提取 geometry，通常不如保持简单结构和提升 dense visual evidence 稳定。
```

## 12. 下一步建议

优先方向：

```text
1. 保持当前主方法简单，不继续堆复杂 scorer。
2. 做多 split / 多 seed，报告 mean ± std。
3. 若要提升，优先尝试更强 frozen evidence：
   - H+768
   - 7B 448
   - 7B 640
4. 如果允许额外轻监督，优先标注少量 placement region bbox/polygon 或 part center。
5. 如果继续改 loss，只做低参数 calibration / hard-negative weighting，并必须跨 split 验证。
```

不建议继续优先做：

```text
d_slot=512
typed_two_stage
attention geometry as bbox
location prior from attention center
current local refinement
current patch evidence
large reranker / large relation MLP
```

一句话总结：

```text
当前最可靠的方法不是训练复杂小模块，而是使用 frozen DINOv3 dense features，
通过轻量 PDDL-conditioned adapter 进行 object binding、placement factorization、
structured ranking 和 legal decoding。
```
