3. **Blocksworld synthetic object-token 实验**
   - 入口：`training/run_bws_structural.py`
   - 当前脚本已经修复为从 problem PDDL 自动读取 blocks/columns。
   - 修复后会检查 state enumeration objects、PDDL compile objects、canonical atom vocabulary 是否一致。
   - 但它仍是 synthetic object-token setting，并且默认 `state_diff` 属于 oracle transition supervision，所以仍不应作为真实视觉或 PDDL weak supervision 正式主结果。

所以当前可保留的严谨表述是：

- **真实视觉路径已经实现了缓存式 pipeline**：图片渲染/读取 -> DINOv3 patch features -> PaQ model。
- **synthetic object-token 路径也实现了**，但它不是端到端视觉实验。
- **用 `state_diff` transition mask 的结果是 oracle transition supervision**，不能说成纯 PDDL/action weak supervision。
- **用 `pddl` transition mask 的结果才是更接近 B-style PDDL 弱监督的设置**，但对 conditional effects 不完整。
- **当前最强真实 cached-DINOv3 数字来自 `state_diff` oracle upper bound**：K=200, full condition, F1=0.9815。

## 2. PDDLDomainCompiler 实现细节

`PDDLDomainCompiler` 的目标是把 PDDL domain 变成模型可以直接使用的结构，而不是手写 label 列表。

输入：

- `domain_path`
- `objects`
- `static_predicates`

示例：

```python
compiler = PDDLDomainCompiler(str(BWS_DOMAIN))
domain_info = compiler.compile(
    objects={"block": ["Y", "P", "R", "O"], "column": ["C1", "C2", "C3", "C4"]},
    static_predicates={"rightof", "leftof"},
)
```

### 2.1 类型和对象

编译器从 unified_planning 读取 PDDL 类型：

- `domain_info.types`
- `domain_info.type_to_idx`

然后根据传入的 `objects` 生成：

- `domain_info.objects`
- `domain_info.obj_name_to_idx`
- `domain_info.obj_type_ids`

模型后续的 object slots 顺序严格跟 `domain_info.objects` 一致。

### 2.2 dynamic predicate schema

编译器遍历 PDDL fluents，过滤掉 static predicates：

```python
if f.name in static:
    continue
```

每个 dynamic predicate 会生成一个 `PredicateSchema`：

- `name`
- `arity`
- `param_types`
- `param_names`
- `action_roles`
- `gloss`

`action_roles` 通过 predicate 是否出现在 action preconditions/effects 中推断。`gloss` 来自 `PREDICATE_GLOSSES` 或启发式生成。

### 2.3 canonical ground atoms

对每个 predicate schema，根据参数类型枚举所有合法 object combinations。

例如 Blocksworld 四个 block、四个 column：

- `clear(block)` -> 4 atoms
- `inColumn(block,column)` -> 16 atoms
- `on(block,block)` -> 12 atoms，过滤自关系
- 总计 32 dynamic atoms

枚举顺序固定：

1. predicate schema 按 name 排序。
2. 每个 predicate 内按 object product 顺序枚举。
3. same-type binary predicate 过滤 `(x,x)`。

这个顺序就是模型输出 `canonical_scores` 的顺序，也是 label vector 的顺序。

### 2.4 action semantics masks

对每个 grounded action instance，编译器生成：

- `precondition_mask`
- `effect_delta`
- `add_mask`
- `del_mask`
- `frame_mask`

其中：

- `effect_delta[i] = +1` 表示 action add 第 i 个 atom。
- `effect_delta[i] = -1` 表示 action delete 第 i 个 atom。
- `effect_delta[i] = 0` 表示未改变。
- `add_mask = effect_delta > 0`
- `del_mask = effect_delta < 0`
- `frame_mask = effect_delta == 0`

注意：对有 conditional effects 的 domain，如果只使用静态 PDDL declaration masks，`add_mask/del_mask/frame_mask` 可能不完整。此时 `state_diff` 或 `pddl_sim` 才能得到实际发生的 add/delete，但这已经更接近 oracle/simulator supervision。

## 3. PaQModel 模型结构

主模型在 `paq/model.py`。

总体结构：

```text
image or feature tokens
  -> visual encoder / cached feature input
  -> object slots
  -> slot type classifier
  -> predicate queries from PDDL schema
  -> predicate slot attention
  -> type-aware scoring head
  -> canonical atom logits
```

### 3.1 输入模式

模型支持两类输入：

1. 图片输入：`images.shape == (B,3,H,W)`
   - 进入 `visual_encoder`。
   - 输出 patch features `(B,N_patches,D)`。

2. feature token 输入：`images.shape == (B,N,D)`
   - 直接认为已经是 visual features。
   - 会经过 `feat_proj`。

当前真实 DINOv3 cached pipeline 使用第二种：从 `features.pt` 读取 `(B,196,256)` patch features。

### 3.2 direct_object_tokens 模式

`PaQModel` 有一个重要参数：

```python
direct_object_tokens: bool = False
```

当 `direct_object_tokens=True` 时：

- 输入 feature token 数必须等于 object slot 数。
- 模型跳过 object slot attention。
- 直接把输入 feature tokens 当 object slots。
- 这是 synthetic object-token runner 使用的模式。

也就是说：

- `run_strips_structural.py` 使用 `direct_object_tokens=True`。
- `run_bws_structural.py` 使用 `direct_object_tokens=True`。
- 这两个 runner 不是端到端视觉 grounding。

真实 DINOv3 few-shot runner 不传 `direct_object_tokens=True`，因此走 object slot attention。

### 3.3 object_slot_init

训练时 `train_condition()` 会用 `_build_object_slot_init()` 初始化 object slots：

- block slots 有颜色 cue。
- column slots 有位置 cue。
- 每个 object slot 有 identity dimension。

这样做的目的：

- scoring head 假设 object slot 顺序和 `domain_info.objects` 对齐。
- 完全随机 slot 初始化会让 object identity 不稳定。
- 这个初始化是一个强 inductive bias，不是纯无监督 object discovery。

### 3.4 PredicateQueryEncoder

`PredicateQueryEncoder` 把 PDDL predicate schema 编成 query embedding。

每个 predicate query 使用：

- predicate name char-LSTM
- arity embedding
- argument type embedding
- action role embedding
- gloss char-LSTM

然后用 gated fusion 合成 `queries: (N_predicates,D)`。

这部分支持“predicate-as-query”的核心设定：模型不是把 predicate 当固定 label ID，而是把 PDDL typed predicate signature 编成 query。

### 3.5 ObjectSlotAttention 和 PredicateSlotAttention

`DualSlotAttention` 包含：

- `ObjectSlotAttention`
  - 输入 patch features。
  - 输出 object slots。
  - 使用 slot attention 迭代更新。

- `SlotTypeClassifier`
  - 对每个 object slot 预测 PDDL type。
  - 训练和评估时，目前 scoring 仍主要使用 oracle `object_type_ids`，以保持 canonical output dimension 稳定。

- `PredicateSlotAttention`
  - 用 predicate queries 去 attend object slots。
  - 输出 predicate slots。

### 3.6 PredicateScoringHead

`PredicateScoringHead` 按 predicate arity 打分，并支持两种 head：

```text
--scoring-head-type legacy
--scoring-head-type film
```

当前真实 cached-DINOv3 oracle upper-bound 主结果使用 `legacy` head。

#### legacy head

- arity 0：
  - `nullary_scorer(p_slot)`
  - 只看 predicate slot，不看 object slots。

- arity 1：
  - 对每个合法 type object，输入 `[p_slot, obj_slot]`。

- arity 2：
  - 对每个合法 object pair，输入 `[p_slot, obj1, obj2, obj1 * obj2]`。
  - same-type binary predicate 过滤 self relation。

输出是 `(B,N_canonical)`，顺序和 `domain_info.canonical_atoms` 一致。

legacy head 的已知表达力瓶颈：

- nullary scorer 不直接看 object slots。
- unary/binary scorer 是按 arity 共享 MLP，不是 predicate-specific scorer。
- 在 TV Assembly 这种大量 binary predicates 的 domain 中，多个 predicate 共享同一个 binary scorer，会产生干扰。

但截至 2026-05-30，legacy head 在真实 cached-DINOv3 + `state_diff` oracle upper-bound setting 中最稳定，K=200 full F1=0.9815。

#### film head

`film` head 是 predicate-conditioned scorer：

```text
score(p, o_i, o_j) =
  out(FiLM_p(MLP([o_i, o_j, o_i * o_j, |o_i - o_j|])))
```

nullary predicate 会读取 object slots 的 attention summary、mean pooling 和 max pooling。

这个设计更符合 predicate-as-query 精神，但当前实验证据显示它比 legacy head 更难优化，暂时作为探索性 ablation，不作为主结果。

## 5. 损失函数

训练核心在 `train_condition()`。

### 5.1 C1: pointwise state supervision

对应 `PredicateStateLoss`。

输入：

- `scores = model(features)["canonical_scores"]`
- `labels = state_labels`

loss：

```text
L_seed = BCEWithLogits(scores, labels)
```

实现中使用 per-atom `pos_weight` 处理 label imbalance。

这部分是强监督：给定图像/feature，直接监督完整 symbolic state vector。

### 5.2 PredicateContrastiveLoss

对应 `PredicateContrastiveLoss`。

它对 predicate slots 和 predicate queries 做 InfoNCE 风格对齐：

- slot -> query
- query -> slot

当前最佳 TV Assembly synthetic 设置里使用：

```bash
--w-contrast 0
```

原因是 contrastive loss 在当前架构下容易干扰 grounding。

### 5.3 C2: ActionEquivarianceLoss

对应 `ActionEquivarianceLoss`。

输入：

- `scores_t`
- `scores_t1`
- `pre_mask`
- `add_mask`
- `del_mask`
- `frame_mask`

包含三项：

1. precondition consistency
   - action precondition atoms 在 `S_t` 应该为真。

2. effect consistency
   - add atoms 在 `S_t1` 应该为真。
   - del atoms 在 `S_t1` 应该为假。

3. frame consistency
   - frame atoms 在 `S_t` 和 `S_t1` 的概率应该相近。

总 loss：

```text
L_equiv = 0.5 * L_pre + 1.0 * L_eff + 0.5 * L_frame
```

外层还会乘 `w_equiv`。

### 5.4 C3: CounterfactualDiscriminabilityLoss

对应 `CounterfactualDiscriminabilityLoss`。

思想：

```text
E(G(I_t), a_true, G(I_t1)) < E(G(I_t), a_false, G(I_t1))
```

其中 energy 包含：

- precondition violation
- add violation
- delete violation
- frame violation

使用 margin ranking：

```text
max(0, margin + E(pos) - E(neg))
```

只有 `condition == "full"` 时启用。

## 6. 四个训练 condition

`run_structural_experiment()` 支持：

### 6.1 static

只用 C1：

```text
L = L_seed + w_contrast * L_contrast + w_type * L_type
```

不使用 transition dataset。

### 6.2 random_pairs

使用随机 state pair，不使用 action structure。

当前实现中 random_pairs 只做弱 frame consistency：

```text
L_frame = mean(abs(probs_t1 - probs_t))
```

它是控制组，用于检查“只是看到更多 pair”是否有帮助。

### 6.3 adjacent

使用真实相邻 transition：

```text
L = C1 + C2
```

即 pointwise state supervision 加 action equivariance。

### 6.4 full

使用真实相邻 transition：

```text
L = C1 + C2 + C3
```

即再加 counterfactual action discrimination。

## 7. transition warmup

`train_condition()` 里有：

```python
transition_active = (epoch + 1) > transition_warmup_epochs
```

作用：

- 前若干 epoch 只训练 C1/state supervision。
- 等 grounding 基本稳定后，再打开 transition/action structure。

当前 TV Assembly synthetic 诊断里，关键设置是：

```bash
--w-contrast 0
--transition-warmup-epochs 20
```

观察到的现象：

- 没有 warmup 时，C2 从 epoch 1 介入，会把模型推向高召回、低精度、超集预测解。
- 加 warmup 后，adjacent/full 的 F1 明显提升。

这说明问题不是简单增大 K，而是 transition loss 加入时机和 loss 组合的问题。

## 8. transition mask source 的含义

这是最容易误报的地方。

### 8.1 `state_diff`

从真实 `S_t` 和 `S_t1` 的 label 差分构造：

```text
add    = S_t1 == 1 and S_t == 0
delete = S_t == 1 and S_t1 == 0
frame  = not add and not delete
```

性质：

- 这是 oracle observed transition difference。
- 它知道训练 transition 两端的真实 symbolic state。
- 不能说成纯 PDDL weak supervision。
- 可作为 upper bound 或 C-style transition-label supervision。

### 8.2 `pddl`

从 compiled PDDL action semantics 的静态 masks 构造：

- `precondition_mask`
- `add_mask`
- `del_mask`
- `frame_mask`

性质：

- 不看当前 `S_t/S_t1` 的实际差分。
- 更接近 B-style PDDL/action weak supervision。
- 对 conditional effects 不完整。

### 8.3 `pddl_conservative`

和 `pddl` 一样使用 static add/delete masks，但禁用 frame mask：

```text
frame_mask = 0
```

用途：

- 避免 conditional effects 下错误 frame 约束。
- 是一个保守诊断设置。

### 8.4 `pddl_sim`

用 simulator 动态模拟 action 得到实际 add/delete/frame。

性质：

- 对 deterministic Blocksworld 近似等价于 `state_diff`。
- 因此不适合当作纯 PDDL weak supervision 报。
- 主要用于诊断。

## 12. 训练流程细节

训练入口统一落到：

```python
run_structural_experiment()
  -> train_condition()
```

### 12.1 初始化

每个 condition 都重新建一个模型：

```python
model = PaQModel.from_domain_info(...)
```

然后用 `_build_object_slot_init()` 初始化 object slots。

如果传 `train_seed`，会重置：

- torch seed
- numpy seed
- cuda seed

这样不同 condition 可复现地对齐初始化。

### 12.2 state batch

每个 epoch 先跑所有 state batches：

1. forward。
2. 取 `canonical_scores`。
3. 算 `L_seed`。
4. 算 `L_contrast`。
5. 算 type loss。
6. backward + optimizer step。

state batch 永远 active。

### 12.3 transition batch

只有以下 condition 使用 transition batch：

- `random_pairs`
- `adjacent`
- `full`

并且必须满足：

```python
(epoch + 1) > transition_warmup_epochs
```

transition batch 会对 `features_t` 和 `features_t1` 各 forward 一次。

然后：

- `random_pairs`：只做整体稳定性约束。
- `adjacent`：做 `ActionEquivarianceLoss`。
- `full`：做 `ActionEquivarianceLoss` + `CounterfactualDiscriminabilityLoss`。

### 12.4 validation 和 threshold

`evaluate(..., threshold=None)` 会在 validation set 上扫阈值：

```text
0.05, 0.10, ..., 0.95
```

选择 validation F1 最优的 threshold。

训练中保存：

- `best_val_f1`
- `best_threshold`
- `best_state`

最后 test 时：

```python
evaluate(model, state_test_ds, threshold=best_threshold)
```

所以 test 没有参与 threshold selection。

### 12.5 指标

evaluation 输出：

- `precision`
- `recall`
- `f1`
- `macro_f1`
- `exact_match`
- `type_acc`
- `threshold`
- `pred_pos_rate`
- `label_pos_rate`
- `avg_prob`

注意：

- `f1` 是 micro F1，按所有样本和 atoms 汇总。
- `exact_match` 要求一个样本的全体 atoms 完全匹配。
- 对 predicate grounding 来说，EM 往往比 micro F1 更严格。

## 14. 当前实现的合规性审计

### 14.1 没有发现直接 test set 训练泄漏

- 先按 state id 切 train/pool、val、test。
- transition dataset 只从 train/pool states 构造。
- threshold 用 val 选。
- final test 只在最后评估。

这部分逻辑基本正规。

### 14.2 但存在强 supervision 和 synthetic 设置必须标注

以下设置不能包装成真实弱监督视觉结果：

- synthetic object-token features。
- `direct_object_tokens=True`。
- `state_diff` transition mask。
- `pddl_sim` transition mask。
- 使用 oracle `object_type_ids` 做 scoring。

### 14.3 当前 evaluation 使用 oracle object types

`evaluate()` 中明确使用：

```python
obj_types = type_ids_tensor.unsqueeze(0).expand(feats.shape[0], -1)
out = model(feats, object_type_ids=obj_types)
```

这保证 canonical dimensions 稳定，但也意味着评估不是完全依赖 predicted slot types。

如果论文要强调 type discovery，需要另写 predicted-type scoring evaluation。

## 15. 当前已知技术问题

### 15.1 scoring head 表达力瓶颈

当前 scorer 按 arity 共享：

- 一个 nullary MLP。
- 一个 unary MLP。
- 一个 binary MLP。

在 TV Assembly 里，多个二元 predicate 共用同一个 binary scorer，容易互相干扰。

建议改进：

1. nullary scorer 输入加入 global object summary：

```text
[predicate_slot, mean(object_slots), max(object_slots)]
```

2. unary/binary scorer 改成 predicate-specific 或 predicate-conditioned：

```text
score = MLP_p(...)
```

或使用 hypernetwork / FiLM conditioning。

### 15.2 synthetic features 太强

`_build_synthetic_features()` 直接把 true atoms 通过 schema vectors 注入 object features。

这会让任务变成：

```text
从人工编码的 object token 解码 symbolic state
```

而不是：

```text
从真实图片识别 objects/relations/state
```

它适合做机制 sanity check，不适合作最终结果。

### 15.3 Blocksworld runner 对象集合历史问题

`training/run_bws_structural.py` 的对象集合不一致问题已经在 2026-05-30 修复：现在从 problem PDDL 自动读取 objects，并检查 enumeration / compile / canonical labels 一致性。

但不要再引用旧的 `bws_k200_warmup_s0` 结果，因为那些结果来自修复前的 synthetic object-token runner。

### 15.4 conditional effects 和 static PDDL masks

Blocksworld `moveBlock` 有 conditional effects。

如果用 `pddl` static masks：

- action add/delete declaration 不能完整表达实际发生的条件效果。

如果用 `state_diff`：

- 监督更准确，但变成 oracle observed diff。

因此报告时必须把二者分开。

## 19. 下一步实现建议

按优先级：

1. **把 PDDL weak supervision 从 static masks 升级到 conditional-effect-aware masks**
   - 当前 `pddl` / `pddl_conservative` 对 Blocksworld conditional effects 不完整，容易全正 collapse。
   - 下一步应实现不读取真实 `S_t1` label 的 PDDL simulator / conditional-effect evaluator。
   - 报告时可分成：
     - `state_diff`: oracle observed transition upper bound。
     - `pddl`: static PDDL declaration weak supervision。
     - `pddl_conservative`: no-frame conservative diagnostic。
     - `pddl_sim`: simulator-based conditional-effect supervision。

2. **补 predicted-type evaluation**
   - 当前 scoring 使用 oracle object types。
   - 如果要声称 type discovery，需要加入 predicted type scoring 或单独 type grounding 指标。

3. **继续研究 FiLM scoring head**
   - `film` head 已实现 predicate-conditioned scorer。
   - 当前真实 cached-DINOv3 oracle upper-bound 仍以 `legacy` head 最稳定。
   - 后续应单独调 schedule/loss，再决定是否替换主结果。

4. **严格 pixel-to-model end-to-end**
   - 当前主结果是 cached DINOv3 features。
   - 若要写端到端，需要 dataloader 读图片，并在 forward 中运行 DINOv3。

5. **补多 seed 正式统计**
   - 当前 2026-05-30 表是 seed=42 split/train 的强结果。
   - 论文表应补 seed=0/1/2 或至少报告 seed 和 split。

## 20. 一句话总结

当前实现已经包含完整的 PDDL-conditioned predicate-as-query 模型、action equivariance 训练、counterfactual loss、真实 DINOv3 feature cache pipeline 和 synthetic structural runner。2026-05-30 的真实 cached-DINOv3 oracle upper-bound 已达到 K=200 full F1=0.9815，但这仍是 `state_diff` oracle transition supervision，不是 PDDL weak supervision，也不是 strict online pixel-to-DINO end-to-end。当前最重要的下一步是把 Blocksworld 的 PDDL weak supervision 做成 conditional-effect-aware，并继续保持 synthetic / cached real visual / oracle transition / PDDL weak supervision 分层报告。
